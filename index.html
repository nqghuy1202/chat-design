Hey Minh! Quick question about the design system we talked about last week.

I'm deciding which color token to use for hover states on card components. Should it match the primary accent, or stay neutral?

9:45 AM
Hey! For card hover states, always use the neutral palette. The accent should only appear on elements that signal a primary action - keeps visual hierarchy clean.
Here's the token I'd use:
css 
.card:hover {
  background: var(--color-surface-hover); /* slate-100 */
  transition: background 0.15s ease;
}

.card:active {
  background: var(--color-surface-pressed); /* slate-200 */
}
